# 엔픽던 전투 시스템 설계

> 기초 전투 구조(속성 피해, 아츠 반응, 방어불능 스택 등 구현된 시스템)는 `엔픽던_구현현황.md` 참조.
> 이 문서는 **피해량 공식 및 공격력(ATK) 시스템 설계 결정사항** 기록.

---

## 1. 공격력(ATK) 개념

기존 SPD는 힘(STR) + 무기 강화치가 피해량을 직접 결정한다.
엔픽던은 이 사이에 **공격력(ATK)** 버퍼를 두어 스킬 계수 설계를 무기 종류와 독립시킨다.

```
STR, 기질 식각, 버프
        ↓
     공격력(ATK)
        ↓
   피해량 계산
```

### ATK 공식

```
ATK = 60 + (STR - 10) × k + 기질 식각 ATK 보너스 + 버프 보너스
```

| 항목 | 값 | 비고 |
|---|---|---|
| 기본 ATK | 60 | STR = 10 기준 시작값. 역전 방지 설계 |
| 능력치 계수 k | `ATK_PER_STAT` | TODO: 수치 확정 |
| 기질 식각 보너스 | 항상 > 10 보장 | 식각 시스템 구현 후 연동 |

> **역전 방지 근거**: 기질 식각이 ATK를 반드시 10 이상 올리도록 설계되어,
> 최솟값(1 + 식각) > 최댓값(ATK×10%) 역전이 구조적으로 불가능.

## 2. 피해량 공식

### 기준값 (한손검 / 스킬)

```
최솟값 = 1 + 식각 수치 + 스탯 보너스
최댓값 = floor(ATK × 10%) + 스탯 보너스
스탯 보너스 = max(0, 현재 STR - 기질 요구 능력치)
```

예시:
- ATK 100, 식각 0, 스탯 보너스 0 → `1 ~ 10` 피해
- ATK 120, 식각 3, 스탯 보너스 2 → `6 ~ 14` 피해

### 스킬 피해

```
스킬 피해 = damageRoll() × SKILL_MULT
```

- `damageRoll()`은 **무기 배율 없는** 기준값 반환
- 무기 종류와 무관하게 동일 계수 → 스킬 밸런스 설계 일관성 확보
- 툴팁: "공격력의 N% 피해" 형식으로 표기 (ATK = 100 → 계수 1.5 → "150의 피해")

### 기본 공격 피해

```
기본 공격 피해 = ((최솟값 ~ 최댓값) + 스탯 보너스) × 무기 배율 × finalAttackEfficiency()
```

무기 배율은 기본 공격에**만** 적용. 스킬은 자체 SKILL_MULT를 사용.

## 4. finalAttackEfficiency()

기본 공격에 적용되는 모든 배율 버프를 **한 메서드에서 집계**하는 구조.

```java
// Hero.java
public float finalAttackEfficiency() {
    float efficiency = 1.0f;
    // 공격 배율 관련 버프 추가 시 여기에만 등록
    return efficiency;
}
```

- 새로운 공격 효율 버프를 추가할 때 이 메서드에만 추가하면 됨
- 스킬 피해에는 미적용 (스킬은 자체 SKILL_MULT 사용)

---

## 5. damageRoll() 분기 구조

`basicAttackMode` 플래그로 기본 공격 / 스킬 분기:

```
damageRoll() 호출
  ├─ basicAttackMode = true  → basicAttackDamageRoll()
  │    (무기 배율 × finalAttackEfficiency() 포함)
  └─ basicAttackMode = false → skillDamageRoll()
       (ATK × 10% 기준, 무기 배율 없음)
```

`onAttackComplete()`에서만 `basicAttackMode = true`로 설정 후 복원.

## 7. 다국어(현지화) 지원

SPD에 `Messages.java` + `.properties` 파일 기반 다국어 시스템이 이미 존재.
일본어(`_ja`), 한국어(`_ko`) 포함 20개 언어 지원 중.

**현재 오퍼레이터 텍스트는 한국어 하드코딩** 상태:
```java
@Override public String name() { return "진천우"; }  // 현재
@Override public String name() { return Messages.get(this, "name"); }  // 다국어 적용 시
```

다국어 대응 시 `assets/messages/operators/` 디렉토리를 새로 만들고
`.properties` 파일에 텍스트를 이전하는 작업 필요.
→ **콘텐츠 완성 이후 현지화 단계에서 처리** 예정.
