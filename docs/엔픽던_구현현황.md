### 1-3. 방어불능 스택 시스템

**파일**: `actors/buffs/DefenselessStack.java`

물리 이상 4종을 `DefenselessStack.apply(enemy, type, attacker)` 단일 진입점으로 관리.

| 종류 | 첫 적용 | 스택 있을 때 |
|------|--------|------------|
| `LAUNCH` (띄우기) | +1스택 | +1스택 + `damageRoll × 1.0` 물리 피해 |
| `KNOCKDOWN` (넘어뜨리기) | +1스택 | +1스택 + `damageRoll × 0.7` 물리 피해. 4스택 도달 시 넉백 1칸 ✅ |
| `HEAVY_ATTACK` (강타) | +1스택 | 전량 소모 + `damageRoll × stacks × 1.5` 물리 피해 |
| `ARMOR_BREAK` (갑옷파괴) | +1스택 | 전량 소모 + `damageRoll × stacks × 0.5` 물리 피해 + `ArmorBreaked` 디버프 |

**쇄빙**: `apply()` 진입 시 대상이 `Frozen` 상태이면 동결 소모 + `damageRoll × 3.0` 대량 물리 피해 → 이후 일반 스택 처리 계속

### 2-2. 연계기 큐 시스템

**파일**: `operators/ChainQueue.java`

#### 타이머 방식
`Actor.now()` 절대 게임 시간 기반. 이속/공속 버프와 무관하게 활성화 시점으로부터 정확히 `DEFAULT_WINDOW(3f)` 게임 시간 후 만료.

#### 큐 순서 규칙

1. **도착 순서** 기반 FIFO — 조건이 먼저 충족된 연계기가 앞
2. **동일 타이밍** 충족 시 파티 순서 우선
3. **이미 큐에 있는 오퍼레이터 재충족** → 현재 위치 유지, 만료 시간만 갱신
4. **만료된 항목** 자동 폐기

#### ⑤ 기존 SPD 시스템 정리 ✅

**방침**: SPD 원본 코드는 최소한으로 수정. 호출부 전면 제거 대신 **`attachTo()` 리다이렉트** 방식으로 세계관 통합.

| 항목 | 처리 방식 | 상태 |
|------|----------|------|
| `Burning` (적에게) | `attachTo()`에서 Hero 아닌 대상 → `Combustion.apply()` 후 `return false` | ✅ |
| `Frost` (적에게) | `attachTo()`에서 Hero 아닌 대상 → `Frozen.apply()` 후 `return false` | ✅ |
| `Corrosion` 표기 | Hero에게는 유지, 텍스트만 "침식"으로 변경 (`.properties` 수정) | ✅ |

- Hero에게는 기존 SPD Burning/Frost/Corrosion 효과 그대로 유지
- 포션·함정 등이 몬스터에 Burning/Frost 적용 시 자동으로 엔드필드 버프로 전환
