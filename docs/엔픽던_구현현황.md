### 1-3. 방어불능 스택 시스템

**파일**: `actors/buffs/DefenselessStack.java`

물리 이상 4종을 `DefenselessStack.apply(enemy, type, attacker)` 단일 진입점으로 관리.

| 종류 | 첫 적용 | 스택 있을 때 |
|------|--------|------------|
| `LAUNCH` (띄우기) | +1스택 | +1스택 + `damageRoll × 1.0` 물리 피해 |
| `KNOCKDOWN` (넘어뜨리기) | +1스택 | +1스택 + `damageRoll × 0.7` 물리 피해. 4스택 도달 시 넉백 1칸(TODO) |
| `HEAVY_ATTACK` (강타) | +1스택 | 전량 소모 + `damageRoll × stacks × 1.5` 물리 피해 |
| `ARMOR_BREAK` (갑옷파괴) | +1스택 | 전량 소모 + `damageRoll × stacks × 0.5` 물리 피해 + `ArmorBreaked` 디버프 |

**쇄빙**: `apply()` 진입 시 대상이 `Frozen` 상태이면 동결 소모 + `damageRoll × 3.0` 대량 물리 피해 → 이후 일반 스택 처리 계속

### 2-2. 연계기 큐 시스템

**파일**: `operators/ChainQueue.java`

#### 타이머 방식
`Actor.now()` 절대 게임 시간 기반. 이속/공속 버프와 무관하게 활성화 시점으로부터 정확히 `DEFAULT_WINDOW(3f)` 게임 시간 후 만료.

```
예) Actor.now() = 2.2에 활성화 → 5.2에 만료
```

#### 큐 순서 규칙

1. **도착 순서** 기반 FIFO — 조건이 먼저 충족된 연계기가 앞
2. **동일 타이밍** 충족 시 파티 순서 우선 (`teamOperators` 순회 순서로 보장)
3. **이미 큐에 있는 오퍼레이터 재충족** → 현재 위치 유지, 만료 시간만 갱신 (순서 변동 없음)
4. **만료된 항목** 자동 폐기

#### 주요 API

| 메서드 | 설명 |
|--------|------|
| `enqueue(op)` | 큐 등록. 이미 있으면 현재 위치에서 타이머 갱신 |
| `tick()` | 매 Hero 행동 완료 시 호출. 만료 항목 폐기 |
| `peek()` | UI 버튼용 헤드 조회 (소비 없음) |
| `consume()` | 발동 시 헤드 제거 및 반환 |

#### Hero.java 연결

| 메서드 | 설명 |
|--------|------|
| `checkChainTriggers(target)` | 이벤트 발생 시 조건 충족 오퍼레이터를 큐에 추가 |
| `activateFrontChain(target)` | UI 버튼 입력 → 큐 헤드 소모 + `activateChain()` + `resetCooldown()` |
