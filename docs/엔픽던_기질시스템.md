# 엔픽던 기질(Trait) 시스템 설계

---

## 1. 개요

기존 SPD의 **무기 아이템** 역할을 **기질**로 대체한다.

| 구분 | SPD | 엔픽던 |
|---|---|---|
| 외형 | 무기를 장착·교체 | 무기는 오퍼레이터 고정 (룩 전용) |
| 능력치/옵션 | 무기 자체에 내장 | **기질**이 담당 |
| 강화 | 무기 강화 (+N) | 기질 **식각** (+N) |
| 저주 | 저주받은 무기 | **침식된 기질** |

### 무기의 역할 분리

- **무기**: 각 오퍼레이터에게 고정 배정. 외형 전용. 게임 중 교체 불가 (특수 상황 제외).
- **기질**: 무기에 장착하는 아이템. 능력치 요구치 / 옵션 / 강화를 모두 담당.

---

## 2. 기질 등급 (티어)

| 티어 | 비고 |
|---|---|
| 1티어 | 초반 장착 가능 |
| 2티어 | |
| 3티어 | |
| 4티어 | |
| 5티어 | 공용 + **특수 기질** 모두 존재 |

각 티어에 맞는 **요구 능력치(STR)** 존재. `traitRequiredStat()`으로 코드에서 참조.

---

## 3. 기질 분류

### 공용 기질

- 티어 1~5 전 구간에 존재
- **모든 오퍼레이터**가 장착 가능
- 특수한 오퍼레이터 고유 효과 없음
- 공통적으로 유용한 일반 옵션 제공 (예시):
  - 공격력 추가 증가
  - 공격 속도 소폭 증가
  - 공격 시 일정 확률로 HP 회복
  - 등

### 특수 기질

- **5티어 전용**
- 오퍼레이터 1명당 3종류 (총 25명 × 3 = 75종)
- **해당 오퍼레이터가 메인일 때** 장착 시 고유 효과 발동
- 다른 오퍼레이터가 장착하면 공용 기질처럼 기본 옵션만 적용

예시 (관리자):
> 오리지늄 결정 특화 기질 → 오리지늄 결정 소모 시 추가 물리 피해

**구현 일정**: 초기에는 오퍼레이터당 1종 구현, 이후 패치로 순차 추가.

### 침식된 기질

- SPD의 **저주받은 무기** 대응
- 상세 설계 미확정 (저주받은 무기 시스템 분석 후 결정)

---

## 4. 기질 식각 (강화)

- SPD의 무기 강화(`+N`) 시스템과 동일 구조
- 식각 수치가 **ATK에 직접 기여** (항상 10 이상 보장 → 피해량 최솟값/최댓값 역전 방지)
- 식각 수치 = `Hero.getEnchantmentLevel()` 반환값 (현재 0 고정, 구현 후 연동)

---

## 5. 코드 구조 계획

SPD의 `Weapon` 클래스를 기반으로 `Trait` 클래스 신규 작성.

```
items/traits/
├── Trait.java              ← 기질 추상 베이스 (tier, requiredStat, enchant)
├── CommonTrait.java        ← 공용 기질 베이스
├── SpecialTrait.java       ← 특수 기질 베이스
├── ErodedTrait.java        ← 침식된 기질
└── special/
    ├── EndministratorTrait1.java  ← 관리자 특수 기질 1번
    └── ...
```

### 특수 기질 효과 정의 패턴

특수 기질 효과는 **오퍼레이터 클래스 안에 정의**하는 방식 권장.
기질 클래스가 공통 뼈대만 갖고, 실제 효과 로직은 오퍼레이터에 위임:

```java
// 예시 구조 (미구현)
public class Endministrator extends Operator {
    @Override
    public void onSpecialTraitEffect(SpecialTrait trait, ...) {
        switch (trait.type) {
            case ORIGINIUM_CRYSTAL:
                // 오리지늄 결정 소모 시 추가 피해
                break;
        }
    }
}
```

이렇게 하면 오퍼레이터 파일 하나에 해당 오퍼레이터의 모든 로직이 집중됨.

## 6. SPD 기존 코드와의 관계

| SPD 요소 | 엔픽던 대응 | 처리 방침 |
|---|---|---|
| `Weapon.java` | `Trait.java`로 대체 | 신규 작성 |
| `Weapon.STRReq` | `Trait.requiredStat()` | 동일 개념 유지 |
| `Weapon.level()` | 기질 식각 수치 | 동일 개념 유지 |
| `KindOfWeapon` | 무기 (룩 전용) | 아이템 시스템 단순화 |
| 저주받은 무기 | 침식된 기질 | 개념 계승, 상세 미확정 |
