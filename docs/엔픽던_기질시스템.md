# 엔픽던 기질(Trait) 시스템 설계

---

## 1. 개요

기존 SPD의 **무기 아이템** 역할을 **기질**로 대체한다.

| 구분 | SPD | 엔픽던 |
|---|---|---|
| 외형 | 무기를 장착·교체 | 무기는 오퍼레이터 고정 (룩 전용) |
| 능력치/옵션 | 무기 자체에 내장 | **기질**이 담당 |
| 강화 | 무기 강화 (+N) | 기질 **식각** (+N) |
| 저주 | 저주받은 무기 | **침식된 기질** |

### 무기의 역할 분리

- **무기**: 각 오퍼레이터에게 고정 배정. 외형 전용. 게임 중 교체 불가 (특수 상황 제외).
- **기질**: 무기에 장착하는 아이템. 능력치 요구치 / 옵션 / 강화를 모두 담당.

## 3. 기질 명칭 체계

기질의 전체 명칭은 티어명 + (수식어) 조합으로 결정된다.

### 무옵션 기질
수식어가 없는 기본 기질. 이름은 티어명만.

```
예) 혼탁 기질 / 안정 기질 / 무결 기질
```

### 공용 수식어 기질
공용 수식어 14종 중 하나가 붙은 기질. 모든 티어에 등장 가능.

```
형식: [티어명] · [수식어]
예)  안정 기질 · 흐름
     세련 기질 · 의료
     혼탁 기질 · 추격
```

전용 수식어가 붙은 기질에는 공용 수식어가 붙지 않는다 (상호 배타).

### 전용 수식어 기질
**무결 기질(5티어) 전용**. 특정 오퍼레이터에게 귀속된 수식어가 붙는다.

```
형식: 무결 기질 · [오퍼레이터명]([수식어])
예)  무결 기질 · 관리자(고통)
     무결 기질 · 관리자(추격)
```

전용 수식어는 공용 수식어와 동일한 단어를 쓸 수 있지만, 효과는 전혀 다르다.

### 침식된 기질
SPD의 저주받은 무기에 대응. 모든 티어에 존재.

```
형식: 침식된 [티어명] 기질
예)  침식된 혼탁 기질
     침식된 무결 기질
```

세부 효과 설계 미확정.

## 5. 식각 (강화)

SPD의 무기 강화(`+N`)에 대응. 식각 수치는 두 가지에 동시 영향:

1. **최소 데미지** 상승 (데미지 하한값 보장)
2. **공격력** 추가 상승

UI 표기 방식 예시 (미확정):
```
무결 기질 +3
공격력 +[기본값+식각 보정] | 최소 +[식각 보정]
```

코드: `Hero.getEnchantmentLevel()` → 기질 식각 수치 반환

## 7. 전용 수식어 (오퍼레이터 전용, 75종)

### 구조

- **5티어 무결 기질 전용**
- 오퍼레이터 25명 × 3종 = **총 75종**
- 전용 수식어와 공용 수식어는 같은 단어를 써도 효과가 다름
- 전용 수식어가 붙으면 공용 수식어는 붙지 않음

### 장착 효과

| 장착자 | 적용 효과 |
|--------|---------|
| 해당 오퍼레이터가 메인 | 기질 기본 효과(공격력 증가) + 전용 효과 |
| 다른 오퍼레이터가 메인 | 기질 기본 효과(공격력 증가)만 |

### 예시 (관리자)

| 기질명 | 전용 효과 |
|--------|---------|
| 무결 기질 · 관리자(고통) | 오리지늄 결정 파괴 피해 증가 |
| 무결 기질 · 관리자(추격) | 궁극기로 오리지늄 결정 소모 시, 즉시 오리지늄 결정 재부착 |
| 무결 기질 · 관리자(?) | TODO |

## 8. 드랍 및 획득 시스템

### 무결 기질 획득 경로

SPD의 5티어 무기와 동일:
- 몬스터 드랍
- 상자 보상
- 상점 판매

세부 드랍처 및 드랍률: 미확정

### 무결 기질 드랍 시 전용 수식어 등장 확률

무결 기질이 드랍될 때, 전용 수식어 포함 여부는 아래 확률로 결정:

```
무결 기질 드랍
 ├─ 70%: 무옵션 또는 공용 수식어 기질
 └─ 30%: 전용 수식어 기질
          ├─ 80%: 현재 메인 오퍼레이터의 전용 기질
          └─ 20%: 다른 오퍼레이터의 전용 기질 (랜덤)

최종 확률 요약:
  무옵션/공용 수식어 무결 기질: 70%
  자신의 전용 기질:             24%  (30% × 80%)
  타인의 전용 기질:              6%  (30% × 20%)
```

## 9. 코드 구조

```
items/traits/
├── Trait.java                      ← 기질 추상 베이스 (tier, requiredStat, enchant)
├── CommonTrait.java                ← 공용 기질 베이스 (티어별 공격력 증가 포함)
├── SpecialTrait.java               ← 전용 기질 베이스 (operatorClass(), isForCurrentMain())
├── ErodedTrait.java                ← 침식된 기질
├── common/
│   ├── GritTrait.java              ← 굳건함 (의료 수식어 계열, 구현됨 ✅)
│   ├── KineticTrait.java           ← 운동 에너지 (골절 수식어 계열, 구현됨 ✅)
│   └── ...                         ← 나머지 12종 미구현
└── special/
    ├── EndministratorResonanceTrait.java ← 관리자 전용 1 (구현됨 ✅)
    └── ...                              ← 나머지 74종 미구현
```

### 전용 기질 효과 정의 패턴

전용 기질의 효과 로직은 오퍼레이터 클래스 안에 위임하는 방식 권장:

```java
// 예시 구조
public class Endministrator extends Operator {
    @Override
    public void onSpecialTraitEffect(SpecialTrait trait, ...) {
        switch (trait.type) {
            case PAIN:    // 고통 수식어
                // 오리지늄 결정 파괴 피해 증가
                break;
            case PURSUIT: // 추격 수식어
                // 오리지늄 결정 소모 시 즉시 재부착
                break;
        }
    }
}
```

이렇게 하면 오퍼레이터 파일 하나에 해당 오퍼레이터의 모든 로직이 집중됨.

## 10. SPD 기존 코드와의 관계

| SPD 요소 | 엔픽던 대응 | 처리 방침 |
|---|---|---|
| `Weapon.java` | `Trait.java`로 대체 | 신규 작성 |
| `Weapon.STRReq` | `Trait.requiredStat()` | 동일 개념 유지 |
| `Weapon.level()` | 기질 식각 수치 | 동일 개념 유지 |
| `KindOfWeapon` | 무기 (룩 전용) | 아이템 시스템 단순화 |
| 저주받은 무기 | 침식된 기질 | 개념 계승, 상세 미확정 |

## 11. 미확정 사항

| 항목 | 비고 |
|------|------|
| 티어별 기본 공격력 수치 | 공격력 공식 설계 후 확정 |
| 식각 시 표기 형식 | UI 설계 시 확정 |
| 공용 수식어 14종 효과 | 추후 개별 결정 |
| 수식어 티어 제한 | 현재 없음, 밸런스 단계에서 추가 검토 |
| 오퍼레이터별 전용 수식어 3종 목록 | 오퍼레이터 확장 시 개별 설계 |
| 침식된 기질 효과 | 미착수 |
| 무결 기질 드랍처 및 드랍률 | 층/몬스터 설계 단계에서 확정 |
| 무옵션/공용 수식어 등장 비율 | 드랍 설계 단계에서 확정 |
